Basic Git Commands

~$ git status

~$ See the difference between your local directory and git repository.

~$ git pull

~$ Grab latest version of files from master repository.

~$ Always do this before making changes, as well as before committing any changes you've made.

~$ git add

~$ Add your changes to be committed to repository.

~$ git add 'filenamehere'
(adds specified doc)

~$ git add -A
(add all changes)

~$ git commit

~$ Commit changes added using git add.

~$ git commit -m 'comment on changes here'

~$ Use esc and enter :wq when committing without message.

~$ git push

~$ Push committed changes into repository.

~$ Always do this before making changes!

CONFLICTS!

A wild conflict has appeared! Don't panick. Someone else made changes to the same file you were editing. The following syntax will show up in your editor:

<<<<<<< HEAD
Code in your local file/changes you made
=======
Code added by someone else
>>>>>>> randomstufforbranchname

Kindly communicate with your fellow group members to decide who's code shall emerge victorious. Or if you are confident in how to resolve the conflict, just go ahead and input the correct code. Note that if you are wrong, we'll never forgive you =). To commit and push the correct changes onto repository, simply delete the conflict syntax and input the correct code, then add the file and commit again.

Branching

It is recommended you create a branch to build your code on. Here is a quick overview of branch commands.

  1. git branch
    Lists all repository branches. The current branch you are in will have a * beside it and likely be green.
  2. git branch newbranch
    Will create a new branch with the name newbranch (or whatever you decide to name it) assuming one does not already exist. Branch contents will be a duplicate of master branch until you go in and make changes.
  3. git checkout branchname
    Use checkout to move into desired branch.
  4. git branch -m newname
    Did you name your branch something stupid like newbranch? To change the name, make sure you are on the branch you want to rename, then use the git branch -m command followed by your desired branch name.
  5. git merge master
    Use merge command to merge your code back into the master branch. Conflicts may arise if changes were made to master that vary from your own edits. See previous section on CONFLICTS!.
  6. git push --set-upstream origin branchname
    To push your new branch onto the repository, you'll need to use the --set-upstream origin command. Just use git push and git will automatically suggest this.

You'll then want to create a pull request for your edits. To do this, go onto the GitHub repository page, select your new branch and find the button in the picture. Take a guess as to which button I'm talking about.

Fill out appropriate info and create the pull request. Wait patiently for group members to review. If you're confident you can just merge it yourself. But that would kind of defeat the whole purpose of this...
